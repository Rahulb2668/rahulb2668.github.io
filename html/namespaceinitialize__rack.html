def initialize_rack.initialize ( )

Method for initializing the slave boxes.

When rack is booted up, the slave boxes is have 4 byte permanent id hard coded to it. Each box is having its own permanent id. For the communication master assigns a temporary communication id of 1byte which is then further used to communication between master and slave. In order to assign this communication id and few other metadata of last state of the slave box this initialize_rack thread is invoked in the begining of the boot up. Its only run once.The rack is made to loading mode first by sending command of (0,9,0) then the box_id and dev_ids are taken from the database and then communication id is broadcasted with ‘boradcast_id’ method and then command is sent to make the led turn red and then back to green.

Note
packet structure is ({0,17,dev_id, box_id,LRC})
