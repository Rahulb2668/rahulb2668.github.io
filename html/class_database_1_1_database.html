Detailed Description

Class containing all the method for data management.

The function in this class is defined to for the data base manipulation. All the metadata published on the broker for a rack is stored in the sql database in the master of the rack. Writing data to the table, reading data from the table etc.. functions of these tasks are defined under this class. Database is database1.db, table name is rack

◆ assign_box_id()

def Database.Database.assign_box_id (   self,
  Id 
)

Method to assign temporary box_id.

The device id is a 4 byte id, so whenever master communicates to the slave, either it has to use this 4 byte id or it can use a unique comm id for addressing the slave, this is done in this method. It takes all the device id and assigns it a unique comm id and updates the database. After this all the communication between master and slave is using this box_id. After box_id is updated the new box_id and device id is broadcasted to the slaves to store the new comm id to the slave box.

Function to check the otp is userotp or delivery otp.

Method is to find the entered otp is loading otp or unloading otp and return flag accordingly. It takes the otp via arguments firstly and try to get all the details with
the otp as ‘deliveryotp’ column in the database , if the data is None, it means the otp is not in the ‘deliveryotp’, so it try to get the data from db with otp as ‘userotp’. If data still None, then otp is invalid, else return the flag.

◆ rrchecker()

def Database.Database.rrchecker (   self)

This method is used to check the food temperature and box present temperature.

This method executes a query to get all the boxes with food temp greater than 45 and is loaded. In this method a loaded box list is created and dictionary with food and temperature is created. This method also calculated the temperature difference and creates a sorted list of boxes having higher temperature difference. This list is passed to the Round Robin worker thread in the Home_Page.py

Returns
It returns a sorted list of temperature and dictionary with box and corresponding food temperature

This method is used to write the data to the database with the dev_id.

This method writes data to specific column in the table for a specific dev_id.This method is called from on_message callback, column dev_id and the value to be updated in the database is passed to the function. In the function it checks whether the dev_id is already in the database if the dev_id is already in the database the value is updated in the database and if the dev_id is not present in the database then the a new row is inserted with new dev_id and the at the same time assign_box_id() method is called which broadcasts the communication id and the dev_id.

Parameters
columnThe column name to which the value has to written
dev_idThe Id of the device for data is updated
valueThe data
