Detailed Description

This is the main function class.

In this class there are specific signals created to each functions like opening unloading window, opening loading window, email log etc... and these signals are connected to specific slots which perform certain tasks. Each of the signals is emited when a mqtt topic published to the broker or when otp is entered via keypad. Functions like initialisation of the rack, roundrobing of the rack, mqtt connection, database linking everything is initialised and called from this class via specific signals.

◆ __init__()

def Home_Page.Home_Window.__init__ (   self)

Constructor of Class Home_Window.

This init method is automatically invoked when an object for the class Home_Window is created. This window initialises the Gui home Screen. Declares some attributes and calls some methods when the object of this class is created. There are few important methods, which are invoked from this constructor when the rack is booted up.

  1. An object for the database class is created
  2. self.Ui() method is called
  3. Object for the initialize_rack() is created
  4. self.connect() method which establishes the communication between rack and the mqtt broker
  5. Updating the rack mode to 'Idle' until an operation command is sent
  6. self.update_passentry function which checks for the keypad entry
  7. The thread for the initialisation of the rack is started with self.initRack.start() call
  8. An object for the round robin thread is created and it starts after 15 mins of rack boots up and if the rack is in idle mode

def Home_Page.Home_Window.accept (   self,
  value 
)

Function to trigger the unloading and loading operation.

This method is the main action method which triggers the loading and unloading action and opens the loading and unloading window. When an otp is passed to the method, it checks whether the otp is for loading operation or for unloading operation ie,userotp or deliveryotp. This is done with check_who function, which returns deliveryflag or userflag according to the type of otp. The otp type is checked in the sql local database.If an otp is passed , it checks whether the otp is AB*#AB if yes it closes the application, if the otp is masterpwd then the signal for the opening setting window is emitted. This settingsignal is connected to a slot which opens the setting window. If the both conditions are wrong then otp is made to integer and passed to the check_who method where the master checks whether the otp is for loading operation or unloading operation. check_who method returns flag for loading operation it returns delivery_flag and for unloading operation user_flag.If the otp is loading otp, then a dictionary with boxno and food name are created and passed to the placefoodwindow.py. Here the box numbers are taken based on the sku_ids and are created such that for one skuid, the corresponding boxes. And if it's a user otp then the user name and food list is taken and passed to the userwindow.py along with boxnos. If the otp is not found in the database an invalid otp message box is displayed.

Parameters
valueOne-Time Password(Either via Mqtt or Keypad)

def Home_Page.Home_Window.Connect (   self)

Method to establish Mqtt connection.

Paho-Mqtt library is used for mqtt connection. A secured tls connection is established. Mosquitto is used as the broker.Last will and testament is also registered. Certain callbacks like on_message(), on_publish(), on_disconnect are defined and connected to specific methods in this method. Client tries to connect to the broker if connection fails the network_label is updated as disconnected and the loop to reconnect is initialised. If connection is sucessfull then the rack is subscribed to the topic 'agile-bot/{Rack_id}/#'

◆ update_passentry()

def Home_Page.Home_Window.update_passentry (   self)

Function to read keypad entry.

This slot is used to read from the file where the keypad entered number is stored and it then set it to the linedit on the Window.This method is called on every 200 miliseconds by the Qtimer method in pyqt. When a keypad is pressed the value is written to mmap file , from which this slot reads the values and if the value is not 'D' the value is appended to the entered_otp and set as text on the lineedit. if the length of the entered_otp exceeds more than 6 then self.accept method is called where the entered_otp is checked its valid.

Note
If the pressed key is 'D' then the last value entered is cleared.
