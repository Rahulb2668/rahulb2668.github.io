Detailed Description

Function to initialize the Rack.

When rack is booted up, the slave boxes is have 4 byte permanent id hard coded to it. Each box is having its own permanent id. For the communication master assigns a temporary communication id of 1byte which is then further used to communication between master and slave. Inorder to assign this communication id and few other metadata of last state of the slave box this initialize_rack thread is invoked in the begining of the boot up. Its only run once.
